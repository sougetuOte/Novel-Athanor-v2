# BUILDING フェーズ ガードレール

## Purpose

BUILDING フェーズにおいて、TDD サイクルを厳守させるガードレール。

## Activation

- `.claude/current-phase.md` が `BUILDING` である
- `/building` コマンド実行後

## MUST（必須）

1. **仕様書の事前確認**: 実装開始前に `docs/specs/` を読み込む
2. **TDD サイクル厳守**:
   - Red: 失敗するテストを先に書く
   - Green: テストを通す最小限のコード
   - Refactor: 設計改善（テストは常にパス）
3. **ドキュメント同期**: コード変更時は対応ドキュメントも同時更新
4. **サイクル完了報告**: 1サイクル完了ごとにユーザーに報告

## MUST NOT（禁止）

1. **仕様なし実装**: 対応仕様書がない状態での実装開始
2. **テストなし実装**: テストを書かずに本実装を進める
3. **ドキュメント未更新**: コードだけ変更してドキュメントを更新しない

## Warning Templates

### 仕様書不在時
```
⚠️ 仕様書警告
対応する仕様書が見つかりません。
1. /planning で仕様書を作成
2. 仕様書のパスを指定
3. 「仕様書なしで続行」と明示（非推奨）
```

### テストスキップ要求時
```
⚠️ TDD警告
テストなしでの実装はルールに反します。
1. テストを先に書く（推奨）
2. テスト観点を列挙してから実装
3. 「テストなしで続行」と明示（非推奨）
```
