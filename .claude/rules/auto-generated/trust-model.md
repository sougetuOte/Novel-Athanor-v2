# 信頼度モデル

TDD 内省パイプラインにおいて、テスト失敗パターンからルール候補を生成するための信頼度モデル。

## 観測回数による段階化

| 段階 | 観測回数 | アクション |
|------|---------|-----------|
| **記録** | 1回 | `docs/memos/tdd-patterns/` にパターン記録ファイルを作成 |
| **注意** | 2回 | 既存パターンの `観測回数` をインクリメント（ユーザー通知なし、ログ更新のみ） |
| **候補生成** | 3回（閾値） | `.claude/rules/auto-generated/draft-NNN.md` にルール候補を生成 |
| **承認待ち** | 3回+ | PM級としてユーザーに承認要求 |

**初期閾値: 3回**。運用データに基づいて調整可能。

## パターン照合ロジック

MVP では手動照合。完全実装での自動照合:

```
1. 新規パターン記録時に既存パターンとの類似度をチェック
   - ファイルパスの一致（同一ファイル or 同一ディレクトリ）
   - エラーメッセージのキーワード一致（テスト名、例外型等）

2. 類似パターンが見つかった場合
   - 既存パターンの `観測回数` をインクリメント
   - `関連パターン` リストに新規観測を追記

3. 閾値到達時
   - ルール候補を draft-NNN.md として生成
   - 候補にはパターンの要約 + 推奨ルール文を含む
   - PM級としてユーザーに承認要求（/pattern-review で確認）
```

## ルール候補のフォーマット

```markdown
# Draft Rule: [ルール名]

**生成日**: YYYY-MM-DD
**観測回数**: N
**ステータス**: draft | approved | rejected

## 根拠パターン

| # | 日付 | ファイル | 失敗内容 |
|---|------|---------|---------|
| 1 | YYYY-MM-DD | src/xxx.py | [要約] |
| 2 | YYYY-MM-DD | src/yyy.py | [要約] |
| 3 | YYYY-MM-DD | src/xxx.py | [要約] |

## 推奨ルール

[ルール文: 「XXX のような変更を行う際は YYY に注意すること」]

## 適用範囲

- 対象ファイルパターン: `src/**/*.py`
- 対象操作: [Edit/Write]
```

## ルール寿命管理

- 各承認済みルールに `last_matched` 日付をメタデータとして記録（ISO 8601形式。PostToolUse hook がテストパターン検出時に自動更新）
- `/daily` 実行時に 90 日以上未使用のルールを棚卸し対象として通知
- ルール削除は **PM級**（人間承認必須）

## 権限等級

- 信頼度モデル自体の変更: **PM級**
- パターン記録の追加: **PG級**（PostToolUse hook が自動記録）
- ルール候補の生成: **PM級**（起票・承認ともに人間承認必須）
- ルール候補の承認・却下: **PM級**
