# フルロード

セッション開始時に前回の状態を復元する。`/quick-save` や `/full-save` で保存した状態を読み込む。

## 1. SESSION_STATE.md を読み込む

プロジェクトルートの `SESSION_STATE.md` を読む。
存在しない場合は「SESSION_STATE.md が見つかりません。新規セッションとして開始します。」と報告して終了。

## 2. プロジェクト状態ファイルを読み込む

`.claude/states/novel-generator.json` を読む。以下を把握:

- 現在の Phase (PLANNING / BUILDING / AUDITING)
- L4 各 Phase の進捗 (completed / pending)
- テスト件数、品質メトリクス
- 関連アーティファクトのパス

## 3. 関連ドキュメントの確認

SESSION_STATE.md の「コンテキスト情報」セクションに記載されたドキュメントのうち、
「次のステップ」の実行に必要なものを特定する。**読み込みはまだ行わない**（コンテキスト節約）。

## 4. 復帰サマリーを報告

以下の形式で報告:

```
--- full-load 完了 ---
前回セッション: YYYY-MM-DD
Phase: [現在のPhase]
進捗: [完了Phase / 全Phase]
テスト: X件、mypy Y、ruff Z

■ 前回の完了タスク (要約):
  - ...

■ 未完了/進行中:
  - ...（なければ「なし」）

■ 次のステップ:
  1. ...
  2. ...

■ 必要に応じて読み込むドキュメント:
  - [ファイルパス]: [何のために必要か]
---
```

## 5. ユーザーの指示を待つ

「次のステップ」をそのまま実行するか、別の作業をするかはユーザーが決める。
ドキュメントの読み込みも、実際に作業を開始するタイミングで行う。
**先回りして大量のファイルを読み込まないこと。**
