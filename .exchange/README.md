# Agent Exchange Protocol

異なるAIエージェント間（Claude Code ↔ Google Antigravity など）でレビューや議論を行うための交換プロトコル。

## 概要

このディレクトリは、直接通信できないAIエージェント同士が「交換日記」方式でやり取りするための場所です。

## ディレクトリ構造

```
.exchange/
├── README.md           # このファイル
├── templates/
│   ├── request.md      # レビュー依頼テンプレート
│   └── response.md     # レビュー回答テンプレート
└── sessions/
    └── (セッションファイル)
```

## 使用方法

### 1. 新しいセッションの開始

`templates/request.md` をコピーして `sessions/` に配置します。

**命名規則**: `YYYY-MM-DD_NNN_<type>.md`

| type | 用途 |
|------|------|
| `code_review` | コード品質レビュー |
| `security` | セキュリティレビュー |
| `architecture` | アーキテクチャ整合性 |
| `refactor` | リファクタリング提案 |
| `general` | その他 |

例: `2025-01-20_001_code_review.md`

### 2. レビュー依頼の記入

テンプレートの各セクションを埋めます：
- **Meta**: セッション情報
- **Scope**: レビュー対象範囲
- **Request**: 具体的な依頼内容
- **Attachments**: 必要に応じてコードや参照情報

### 3. 回答の追記

回答側エージェントは、同じファイル内の `---` 区切り以降に回答を追記します。

### 4. フォローアップ（任意）

追加の質問や議論がある場合は、さらに `---` 区切りで追記します。

## プロトコル規約

1. **1セッション = 1ファイル**: 1つのレビュー/議論につき1ファイル
2. **追記のみ**: 相手の記述を編集しない（追記のみ）
3. **往復回数**: 基本1〜2往復で完結させる
4. **完了マーク**: やり取り完了時は Meta セクションの Status を `completed` に変更

## 移植方法

このディレクトリ（`.exchange/`）をそのまま別プロジェクトにコピーすれば利用可能です。
`sessions/` 内のファイルは移植不要（プロジェクト固有のため）。
